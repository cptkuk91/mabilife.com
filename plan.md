## 2. 향후 작업 계획 (Next Steps)

### 2.1. 우선 순위 (High Priority) - 기능 및 알림
- [ ] **알림 시스템 (Notification System)**
    - [ ] 상단 네비게이션 알림 아이콘 (Bell)
    - [ ] 알림 드롭다운/모달 UI
    - [ ] **알림 트리거 구현**:
        - [ ] 내 글에 댓글이 달렸을 때
        - [ ] 내 글에 좋아요가 눌렸을 때
        - [ ] 북마크한 글에 업데이트가 있을 때 (선택 사항)

- [ ] **마이페이지 고도화 (`/profile` or `/mypage`)**
    - [ ] **대시보드**: 내 활동 요약 (작성 글 수, 받은 좋아요 수 등)
    - [ ] **북마크/스크랩**: 내가 저장한 공략/글 모아보기
    - [ ] **작성 글 관리**: 내가 쓴 글 목록 및 수정/삭제
    - [ ] **설정**: 프로필 이미지 변경, 닉네임 변경

- [ ] **상호작용 로직 (Interaction Logic)**
    - [ ] **댓글 (Comment)**: 작성, 수정, 삭제, 대댓글(Reply) 기능
    - [ ] **좋아요 (Like)**: 게시글/댓글 좋아요 토글 및 카운트 실시간 반영
    - [ ] **북마크 (Bookmark)**: 게시글 저장 및 해제 기능

### 2.2. 중기 계획 (Medium Priority)
- [ ] **글쓰기 기능 고도화 (`/write`)**
    - [ ] 이미지 업로드 (Drag & Drop)
    - [ ] 텍스트 에디터 (Bold, Italic, Link 등)
    - [ ] 임시 저장 기능

### 2.3. 백엔드 및 최적화 (Backend & Optimization)
- [ ] **데이터베이스 연동**
    - [ ] MongoDB/Supabase 등 DB 스키마 확정
    - [ ] 유저 인증 (Auth) 세션 관리
- [ ] **SEO 및 성능 최적화**
    - [ ] 메타 태그 동적 생성
    - [ ] 이미지 최적화 (Next/Image)

---

## 3. 제안 사항 (Suggestion)
- **알림 시스템**은 유저 리텐션(재방문)에 중요하므로, 좋아요/댓글 기능과 함께 묶어서 개발하는 것이 효율적입니다.
- **마이페이지**에서 북마크한 글을 쉽게 볼 수 있게 하면 공략 사이트로서의 가치가 높아집니다.
